SQL Quick Reference

Interview Prep
How to approach any SQL question
Spend 60 seconds planning before writing a single line. This saves 3 minutes of rewriting.
The 60 second mental checklist
1
Count the tables and map connections

How many tables? Which columns link them? Draw the chain: Sales → Employees → Departments

2
Read the output columns first

What does the result need? Each column tells you which table and what operation (SUM, LAG, RANK etc.)

3
Map requirements to SQL concepts

"For each X" → PARTITION BY. "Top N" → DENSE_RANK. "Compare previous" → LAG. "If condition" → CASE WHEN.

4
Plan your CTEs

How many layers? CTE1 = joins + aggregation. CTE2 = window functions. Final = filter + sort.

5
Write CTE skeleton first, then fill

Write WITH cte AS ( ... ) SELECT ... first. Fill in columns after structure is correct.

CTE skeleton — write this first every time
WITH step1 AS (
  -- joins + aggregation here
  SELECT ...
  FROM ...
  JOIN ... ON ...
  GROUP BY ...
),
step2 AS (
  -- window functions on top of step1
  SELECT *, DENSE_RANK() OVER (PARTITION BY ... ORDER BY ...) AS rnk
  FROM step1
)
SELECT ...
FROM step2
WHERE rnk <= 2
ORDER BY ...

Inner join
SELECT e.name, d.dept_name, s.salary
FROM Employees e
JOIN Departments d ON e.dept_id = d.dept_id
JOIN Salaries s ON e.emp_id = s.emp_id
Left join + anti-join (find unmatched)
-- All customers including those with no orders
SELECT c.name, o.order_id
FROM Customers c
LEFT JOIN Orders o ON c.id = o.customer_id

-- Customers with NO orders (anti-join)
SELECT c.name
FROM Customers c
LEFT JOIN Orders o ON c.id = o.customer_id
WHERE o.customer_id IS NULL
Self join — employee and manager
SELECT e.name AS employee, m.name AS manager
FROM Employees e
LEFT JOIN Employees m ON e.manager_id = m.emp_id
-- LEFT JOIN so employees with no manager still appear
Cross join — tournament schedule
WITH numbered AS (
  SELECT name, ROW_NUMBER() OVER (ORDER BY name) AS id
  FROM Countries
)
SELECT a.name AS team1, b.name AS team2
FROM numbered a
CROSS JOIN numbered b
WHERE a.id < b.id        -- play once (no reverse duplicates)
-- WHERE a.id != b.id   -- home & away (play twice)
Trap Left join becomes inner join
-- WRONG: WHERE on right table kills NULLs → inner join
LEFT JOIN Dept d ON e.dept_id = d.id
WHERE d.name = 'IT'  -- ❌ unmatched employees disappear

-- RIGHT: move condition into ON clause
LEFT JOIN Dept d ON e.dept_id = d.id AND d.name = 'IT'  -- ✅
Group by / Having / Order by / Limit
Full structure
SELECT department, COUNT(*) AS emp_count, AVG(salary) AS avg_sal
FROM Employees
WHERE salary > 20000          -- filter BEFORE grouping
GROUP BY department
HAVING AVG(salary) > 50000   -- filter AFTER grouping
ORDER BY avg_sal DESC
LIMIT 5
OFFSET 2                      -- skip first 2 rows (page 2)
Execution order — memorize this
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT → OFFSET
This is why you can't use aggregate functions in WHERE — grouping hasn't happened yet.
WHERE vs HAVING
Use WHERE forUse HAVING for
Filter on raw columns before groupingFilter on COUNT / SUM / AVG after grouping
salary > 50000AVG(salary) > 50000
department = 'IT'COUNT(*) > 5
-- ERROR: aggregate not allowed in WHERE
WHERE COUNT(*) > 5   -- ❌
HAVING COUNT(*) > 5  -- ✅
GROUP BY rule — every non-aggregate in SELECT must be in GROUP BY
SELECT dept, name, AVG(salary)  -- dept and name are non-aggregate
FROM Employees
GROUP BY dept, name             -- both must appear here
-- AVG(salary) is aggregate → does NOT go in GROUP BY

Subqueries
Subquery in WHERE
Use when you need to filter based on a calculated value
-- Employees earning above company average
SELECT name, salary
FROM Employees
WHERE salary > (
  SELECT AVG(salary) FROM Employees
)
Subquery in FROM (inline view)
Treat result of a query as a temporary table
SELECT dept, avg_sal
FROM (
  SELECT department AS dept, AVG(salary) AS avg_sal
  FROM Employees
  GROUP BY department
) AS dept_avg
WHERE avg_sal > 60000
Subquery in SELECT
Add a calculated value as a column alongside each row
SELECT
  name,
  salary,
  (SELECT AVG(salary) FROM Employees) AS company_avg,
  salary - (SELECT AVG(salary) FROM Employees) AS diff_from_avg
FROM Employees
Correlated subquery — runs once per row
References the outer query — slower but powerful
-- Employees earning above their own department average
SELECT e1.name, e1.department, e1.salary
FROM Employees e1
WHERE e1.salary > (
  SELECT AVG(e2.salary)
  FROM Employees e2
  WHERE e2.department = e1.department  -- references outer query
)
EXISTS — check if subquery returns any rows
-- Customers who have placed at least one order
SELECT c.name
FROM Customers c
WHERE EXISTS (
  SELECT 1 FROM Orders o WHERE o.customer_id = c.id
)

-- NOT EXISTS — customers with no orders
WHERE NOT EXISTS (...)

CTE (Common Table Expression)
Single CTE
WITH dept_avg AS (
  SELECT department, AVG(salary) AS avg_sal
  FROM Employees
  GROUP BY department
)
SELECT e.name, e.salary, d.avg_sal
FROM Employees e
JOIN dept_avg d ON e.department = d.department
WHERE e.salary > d.avg_sal
Chained CTEs — multi-table joins
WITH step1 AS (
  SELECT a.emp_id, a.name, b.salary
  FROM Employees a
  JOIN Salaries b ON a.emp_id = b.emp_id
),
step2 AS (
  SELECT s.emp_id, s.name, s.salary, d.dept_name
  FROM step1 s
  JOIN Departments d ON s.emp_id = d.emp_id
),
step3 AS (
  SELECT s.*, l.city
  FROM step2 s
  LEFT JOIN Locations l ON s.emp_id = l.emp_id
)
SELECT * FROM step3
ORDER BY salary DESC
CTE + window function filter pattern
You CANNOT filter on window function results in the same query — always use CTE first
WITH ranked AS (
  SELECT *,
    DENSE_RANK() OVER (PARTITION BY dept ORDER BY salary DESC) AS rnk
  FROM Employees
)
SELECT name, dept, salary
FROM ranked
WHERE rnk <= 3  -- filter AFTER window function in CTE

Window function — OVER structure
The OVER skeleton
FUNCTION() OVER (
  PARTITION BY col     -- split into groups, reset per group
  ORDER BY col         -- order rows within each group
  ROWS BETWEEN ...     -- only for rolling calculations
)
What each part does
PartWhat it doesNeeded for
PARTITION BYResets function per group (like GROUP BY but rows stay separate)ROW_NUMBER, RANK, LAG, Rolling
ORDER BYOrders rows within each partitionROW_NUMBER, RANK, LAG, Rolling
ROWS BETWEENControls how many rows to include in calculationRolling totals/averages only
Key rule
"For each X" in the question → PARTITION BY X. Always.
-- "for each customer, find their orders in order"
ROW_NUMBER() OVER (PARTITION BY customer_id ORDER BY order_date)

-- "across entire table" — no PARTITION BY needed
RANK() OVER (ORDER BY salary DESC)

Nth row per group — ROW_NUMBER pattern
WITH ranked AS (
  SELECT customer_id, order_id, order_date,
    ROW_NUMBER() OVER (
      PARTITION BY customer_id
      ORDER BY order_date
    ) AS rn
  FROM Orders
)
SELECT customer_id, order_id, order_date
FROM ranked
WHERE rn = 2   -- second order per customer
Top N per group — DENSE_RANK pattern
WITH ranked AS (
  SELECT department, emp_name, salary,
    DENSE_RANK() OVER (
      PARTITION BY department
      ORDER BY salary DESC
    ) AS rnk
  FROM Employees
)
SELECT department, emp_name, salary
FROM ranked
WHERE rnk <= 3   -- top 3 salaries per department (handles ties)
NTILE — divide into buckets
Split employees into 4 equal salary bands (quartiles)
SELECT name, salary,
  NTILE(4) OVER (ORDER BY salary DESC) AS quartile
FROM Employees
-- quartile 1 = top 25%, quartile 4 = bottom 25%

LAG / LEAD
Concept
LAG = look at the previous row's value. LEAD = look at the next row's value. Like standing in a queue — LAG is behind you, LEAD is in front.
Syntax
LAG(column, n, default_if_null)  OVER (PARTITION BY ... ORDER BY ...)
LEAD(column, n, default_if_null) OVER (PARTITION BY ... ORDER BY ...)

-- n = rows back/forward (default 1)
-- default = what to show instead of NULL (optional)
-- Example: LAG(revenue, 1, 0) shows 0 instead of NULL for first row
Week on week change pattern
WITH weekly AS (
  SELECT
    DATEPART(week, sale_date) AS week_num,
    SUM(revenue) AS weekly_revenue
  FROM Sales
  GROUP BY DATEPART(week, sale_date)
)
SELECT
  week_num,
  weekly_revenue,
  LAG(weekly_revenue, 1) OVER (ORDER BY week_num) AS prev_week,
  weekly_revenue - LAG(weekly_revenue, 1) OVER (ORDER BY week_num) AS wow_change,
  ROUND(100.0 * (weekly_revenue - LAG(weekly_revenue,1) OVER (ORDER BY week_num))
    / LAG(weekly_revenue,1) OVER (ORDER BY week_num), 2) AS wow_pct
FROM weekly
Key behaviors
ScenarioResult
First row with LAGNULL — no previous row exists
Last row with LEADNULL — no next row exists
LAG(col, 7) on daily dataSame day last week
LAG(col, 2)Value 2 rows back — skips one row
No PARTITION BYLooks across all rows mixed together (usually wrong)
Avoid NULL in calculationsUse COALESCE(LAG(col,1), 0)
Rolling totals / Moving averages
ROWS BETWEEN reference
KeywordMeaning
UNBOUNDED PRECEDINGFrom the very first row in partition
N PRECEDINGN rows back from current row
CURRENT ROWThe current row itself
N FOLLOWINGN rows forward from current row
UNBOUNDED FOLLOWINGUntil the last row in partition
Running total
SELECT order_date, amount,
  SUM(amount) OVER (
    PARTITION BY customer_id
    ORDER BY order_date
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS running_total
FROM Orders
Moving average — last 3 rows
SELECT sale_date, revenue,
  AVG(revenue) OVER (
    PARTITION BY product_id
    ORDER BY sale_date
    ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
  ) AS moving_avg_3
FROM Sales
-- 2 PRECEDING = 2 rows back + current = 3 rows total

CASE WHEN
Basic structure
SELECT name,
  CASE
    WHEN score IS NULL THEN 'Unknown'   -- NULL check FIRST always
    WHEN score >= 90 THEN 'A'
    WHEN score BETWEEN 75 AND 89 THEN 'B'
    WHEN score BETWEEN 60 AND 74 THEN 'C'
    ELSE 'Fail'
  END AS grade    -- END and AS are required
FROM Students
Conditional aggregation — CASE inside COUNT/SUM
Pivot row values into separate columns without a PIVOT keyword
SELECT department,
  COUNT(CASE WHEN salary > 50000 THEN 1 END) AS high_earners,
  COUNT(CASE WHEN salary <= 50000 THEN 1 END) AS low_earners,
  SUM(CASE WHEN gender = 'M' THEN salary ELSE 0 END) AS male_payroll,
  COUNT(CASE WHEN status = 'active' THEN 1 END) AS active_count
FROM Employees
GROUP BY department
CASE inside window function
SELECT page_id, visit_date, visitors,
  LEAD(visitors,1) OVER (PARTITION BY page_id ORDER BY visit_date) AS next_day,
  CASE
    WHEN LEAD(visitors,1) OVER (PARTITION BY page_id ORDER BY visit_date) > visitors
    THEN 'Increasing'
    ELSE 'Dropping/Same'
  END AS trend
FROM WebTraffic

NULL handling / COALESCE
NULL rules — all return NULL
NULL = NULL    -- NULL, not True!
NULL != NULL   -- NULL
NULL + 5       -- NULL
NULL > 5       -- NULL
NULL AND TRUE  -- NULL

-- NEVER use = NULL or != NULL in WHERE
WHERE salary = NULL    -- ❌ returns ZERO rows
WHERE salary != NULL   -- ❌ returns ZERO rows

-- ALWAYS use IS NULL / IS NOT NULL
WHERE salary IS NULL       -- ✅
WHERE salary IS NOT NULL   -- ✅
COALESCE — replace NULL with default
-- Replace NULL with 0 (prevents broken math)
SELECT salary + COALESCE(bonus, 0) AS total_comp
FROM Employees

-- Display replacement
SELECT COALESCE(middle_name, 'N/A') AS middle_name

-- Multiple fallbacks — returns first non-NULL left to right
SELECT COALESCE(phone, mobile, email, 'No contact') AS contact

-- Avoid NULL in LAG result
COALESCE(LAG(revenue, 1) OVER (ORDER BY week), 0) AS prev_week
COUNT(*) vs COUNT(column)
COUNT(*)        -- counts ALL rows including NULLs
COUNT(salary)   -- counts only non-NULL values

-- Find number of NULLs in a column:
SELECT COUNT(*) - COUNT(salary) AS null_count
FROM Employees
NULLIF — return NULL if two values are equal
Useful to prevent division by zero
-- Without NULLIF — division by zero error if total = 0
SELECT revenue / total_orders  -- ❌ error if total_orders = 0

-- With NULLIF — returns NULL instead of error
SELECT revenue / NULLIF(total_orders, 0) AS avg_order_value  -- ✅
UNION / INTERSECT / EXCEPT
Structure
SELECT col1, col2 FROM table1
UNION        -- combines, removes duplicates
UNION ALL    -- combines, keeps duplicates (faster)
INTERSECT    -- only rows in BOTH tables
EXCEPT       -- rows in first but NOT in second
SELECT col1, col2 FROM table2
Rules
RuleDetail
Same column countBoth SELECTs must have same number of columns
Matching data typesColumn types must be compatible
Column names from first SELECTAlias in first SELECT sets output column names
ORDER BY goes at endOnly one ORDER BY — applies to final result
Practical example — combine active and VIP customers
SELECT customer_id, name, 'active' AS source FROM ActiveCustomers
UNION ALL
SELECT customer_id, name, 'vip' AS source FROM VIPCustomers
ORDER BY name
INNER JOIN + UNION = redundant
-- INNER JOIN rows are already inside LEFT JOIN results
-- So this combo is pointless — just use LEFT JOIN alone
SELECT ... FROM A INNER JOIN B ...
UNION
SELECT ... FROM A LEFT JOIN B ...
-- Result = same as LEFT JOIN alone

Common interview patterns
Pattern 1 — employee above department average
WITH dept_avg AS (
  SELECT department, AVG(salary) AS avg_sal
  FROM Employees
  GROUP BY department
)
SELECT e.name, e.department, e.salary, d.avg_sal
FROM Employees e
JOIN dept_avg d ON e.department = d.department
WHERE e.salary > d.avg_sal
ORDER BY e.department, e.salary DESC
Pattern 2 — find duplicates
-- Find emails that appear more than once
SELECT email, COUNT(*) AS occurrences
FROM Users
GROUP BY email
HAVING COUNT(*) > 1

-- Get the actual duplicate rows
WITH dupes AS (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY email ORDER BY created_at) AS rn
  FROM Users
)
SELECT * FROM dupes WHERE rn > 1  -- keep only duplicates
Pattern 3 — running total with LAG comparison
WITH monthly AS (
  SELECT
    YEAR(order_date) AS yr,
    MONTH(order_date) AS mth,
    SUM(amount) AS revenue
  FROM Orders
  GROUP BY YEAR(order_date), MONTH(order_date)
)
SELECT yr, mth, revenue,
  SUM(revenue) OVER (PARTITION BY yr ORDER BY mth
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) AS ytd_total,
  LAG(revenue,1) OVER (ORDER BY yr, mth) AS prev_month,
  revenue - LAG(revenue,1) OVER (ORDER BY yr, mth) AS mom_change
FROM monthly
Pattern 4 — top N per group with LAG
WITH totals AS (
  SELECT dept, emp_name, SUM(amount) AS total_sales
  FROM Sales s
  JOIN Employees e ON s.emp_id = e.emp_id
  GROUP BY dept, emp_name
),
ranked AS (
  SELECT *,
    DENSE_RANK() OVER (PARTITION BY dept ORDER BY total_sales DESC) AS rnk,
    LAG(total_sales) OVER (PARTITION BY dept ORDER BY total_sales DESC) AS prev_total
  FROM totals
)
SELECT dept, emp_name, total_sales,
  COALESCE(prev_total, 0) AS prev_total,
  total_sales - COALESCE(prev_total, 0) AS difference
FROM ranked
WHERE rnk <= 2
ORDER BY dept, total_sales DESC

Function structure

    -- logic here

Second largest in array
def second_largest(arr):
    first = second = float('-inf')
    for i in arr:
        if i > first:
            second = first
            first = i
        elif first > i > second:
            second = i
    return second
Palindrome check
def is_palindrome(s):
    return s == s[::-1]   -- s[::-1] reverses the string
Count vowels
def count_vowels(s):
    vowels = ('a','e','i','o','u')
    count = 0
    for i in s:
        if i.lower() in vowels:
            count += 1
    return count   -- return AFTER loop, not inside
Factorial
def factorial(n):
    result = 1
    for i in range(1, n+1):  -- n+1 to include n itself
        result *= i
    return result
Lambda — one line function
-- Regular function vs lambda
def add(x, y): return x + y
add = lambda x, y: x + y   -- same thing

-- Ternary inside lambda
check = lambda x: 'Even' if x % 2 == 0 else 'Odd'
larger = lambda x, y: x if x > y else y
square = lambda x: x * x
map / filter / sorted
numbers = [1,2,3,4,5,6]

-- map: transform every element
list(map(lambda x: x*2, numbers))       -- [2,4,6,8,10,12]

-- filter: keep only matching elements
list(filter(lambda x: x % 2 == 0, numbers)) -- [2,4,6]

-- sorted: sort by condition
students = [("Raj",92),("Priya",78),("Nishanth",85)]
sorted(students, key=lambda x: x[1], reverse=True)
-- [("Raj",92),("Nishanth",85),("Priya",78)]

Numpy basics
import numpy as np

arr = np.array([1,2,3,4,5])

-- Basic stats
np.mean(arr)     -- average
np.sum(arr)      -- sum
np.max(arr)      -- max value
np.min(arr)      -- min value
np.std(arr)      -- standard deviation

-- Boolean masking
arr[arr > 3]      -- [4, 5] — elements greater than 3
arr[arr % 2 == 0] -- [2, 4] — even elements only

-- np.where — conditional replacement
np.where(arr > 3, 'big', 'small')
-- ['small','small','small','big','big']
Quick decision cheat sheet
"For each X"PARTITION BY X
"Top N per group"DENSE_RANK <= N
"Nth row only"ROW_NUMBER = N
"Compare to previous"LAG
"Compare to next"LEAD
"Running total"SUM OVER UNBOUNDED PRECEDING
"Moving avg last N"AVG OVER N-1 PRECEDING
"If this then that"CASE WHEN ... END
"Replace NULL"COALESCE(col, default)
"Filter on aggregate"HAVING
"Filter on raw rows"WHERE
"Every combination"CROSS JOIN + a.id < b.id
"Combine two queries"UNION / UNION ALL
"Rows in both tables"INTERSECT
"In A but not B"EXCEPT
"All left + matched right"LEFT JOIN
"Only matched rows"INNER JOIN
"Unmatched rows only"LEFT JOIN WHERE b.id IS NULL
"Break complex query"CTE (WITH ... AS)
"Ties matter"DENSE_RANK not ROW_NUMBER
"Division by zero risk"NULLIF(col, 0)
"Percent change"100.0 * (new - old) / old
"String pattern match"LIKE '%pattern%'
"Days between dates"DATEDIFF(day, start, end)
Execution order — always
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT → OFFSET
